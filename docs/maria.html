BOBO

Гид по экранам

Для дизайна — что делает каждый экран и какие на нём элементы

Приложение — на Next.js + React + Tailwind CSS; дизайн задаётся классами Tailwind прямо в разметке компонентов. Техническая настройка, запуск и карта «экран → файл» — в файле ФУНКЦИОНАЛ.md в репозитории (его удобно открыть в Claude — он подскажет, где что править). Фирменный цвет — тёплый коричневый «bobo».

Общее

  • Вход — логотип BOBO, поля логин/пароль, подсказка с демо-аккаунтами.
  • Шапка (на всех экранах) — «BOBO» + бейдж роли, имя пользователя, кнопка «Log out».
  • Дашборд роли — кнопки-ссылки на разделы + карточки функций. У пекарни и кафе на кнопке «Requests» — бейдж с числом необработанных запросов.

Экраны администратора

Каталог продуктов

также доступен пекарне при делегировании

Таблица по категориям; добавление/редактирование позиции (название, категория, единица, цена, флаг примечания, дедлайн-часы), архив. Цены показаны как «£X.XX», непроставленные помечаются.

Аккаунты

Таблица пользователей; создание (логин, пароль, роль, привязка к кафе), редактирование, сброс пароля, вкл/выкл.

Кафе

Список кафе; создание/редактирование (название, адрес, контакты), вкл/выкл. Все кафе тарифицируются еженедельно.

Делегирование и настройки

Два переключателя: «пекарня может править каталог» и «…цены». Отображение валюты.

Экраны пекарни

Недельный заказ ★ (главный экран)

самый насыщенный — приоритет для дизайна

  • Вкладки кафе сверху; вкладка подсвечивается (точка + счётчик), если у кафе есть необработанный запрос.
  • Навигатор недель (← неделя →, «This week») и бейдж статуса: Draft / Changes requested / Confirmed; пометка фазы недели.
  • Таблица «продукты × 7 дней»: по умолчанию — только числа, без лишних подписей. Галка «Only ordered items» прячет нулевые строки.
  • Редактирование по кнопке: «Edit» (черновик) или «Request changes» (подтверждённая неделя) → ячейки становятся полями ввода; дни после катоффа заблокированы.
  • Действия: Save, Confirm/Submit (кафе), Accept/Reject (пекарня); список заявок на изменение под таблицей.

Запросы (Requests)

Входящие, ждущие решения: правки кафе к недельным заказам + заявки на изменение доставок. Кнопки Accept/Reject и переход в заказ.

Диспетчеризация (Dispatch)

Выбор даты + чипсы дней недели с числом кафе, у которых есть заказ. По каждому кафе — состав и назначение курьера; статусы доставки.

Инциденты

Разделы «На рассмотрении» и «История»; карточка инцидента (кафе, дата, тип, количество, кто сообщил, фото) с кнопками Confirm/Reject.

Счета (Invoices)

Навигатор недель, кнопка «сгенерировать по всем кафе» (с гейтами), сводка батча, список счетов, «Send all», просмотр счёта с позициями и исключёнными инцидентами.

Экраны кафе

Недельный заказ

Тот же экран, что у пекарни (без вкладок): просмотр числами, «Edit»/«Request changes», подтверждение/отправка.

Запросы

Недельные заказы, ждущие подтверждения кафе, и статусы их заявок на изменение.

Тикеты доставок

Список доставок (новые сверху): статус, состав, фото-подтверждение, кнопка «сообщить об инциденте».

Счета

Полученные счета с позициями и итогом.

Экраны курьера

Маршрут на день

Выбор даты; «принять партию у пекарни»; по каждому кафе — состав, загрузка фото и «доставлено»; кнопка инцидента.
Полная карта «экран → файл», стек и инструкция локального запуска — в ФУНКЦИОНАЛ.md репозитория github.com/KNSTNTN24/bobo. ★ — экран недельного заказа (src/components/OrdersScreen.tsx) — основной для дизайна.
